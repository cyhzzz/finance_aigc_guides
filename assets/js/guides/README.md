## ✅ 内容编辑注意事项

### 1. 转义字符

在JS字符串中需要转义的特殊字符：
- 反斜杠：`\\` → `\\\\`
- 反引号：\` → `\\``
- 例如：`C:\\Users\\` → `C:\\\\Users\\\\`

### 2. 代码块

在Markdown中使用代码块时，建议用不同的格式：

- **单行代码**：用单个反引号 \`代码\`
- **多行代码**：用三个反引号 \`\`\`代码\`\`\`

### 3. 换行

在模板字符串中直接换行即可，无需特殊处理。

### 4. 注释

每个JS文件顶部都有注释说明，包括：
- 操作指引名称
- 更新日期
- 编辑建议

## 🎨 Markdown支持

支持以下Markdown语法：

- ✓ 标题（# ## ###）
- ✓ 粗体（**text**）
- ✓ 斜体（*text*）
- ✓ 列表（有序、无序）
- ✓ 代码块（单个反引号或三个反引号）
- ✓ 表格
- ✓ 引用块（>）
- ✓ 横线（---）
- ✓ 链接（[text](url)）

## 🔄 更新流程

1. **编辑** → 打开对应的JS文件
2. **修改** → 修改模板字符串中的Markdown内容
3. **保存** → 保存文件（Ctrl+S）
4. **刷新** → 刷新浏览器（F5或Ctrl+R）

## 📌 最佳实践

1. **定期备份**：修改前先备份原文件
2. **小步更新**：每次修改一小部分，及时测试
3. **使用版本控制**：建议使用Git管理版本
4. **添加注释**：重要修改添加注释说明
5. **保持格式**：保持与其他文件一致的格式

## 🆘 常见问题

**Q: 修改后页面没更新？**

A: 清除浏览器缓存（Ctrl+Shift+Delete）或强制刷新（Ctrl+F5）

**Q: 语法错误怎么办？**

A: 检查：
1. 模板字符串是否正确（前后都有反引号）
2. 是否有未转义的特殊字符
3. 浏览器控制台是否有错误信息

**Q: 内容太长不好编辑？**

A: 建议：
1. 使用支持Markdown的编辑器（如VS Code）
2. 先在MD文件中编辑，再复制到JS文件
3. 将长内容分段，使用子标题
